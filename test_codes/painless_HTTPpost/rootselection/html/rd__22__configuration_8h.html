◆ timerDelayRoot

#define timerDelayRoot   40000

Reset the node after this interval.

When a node doesn't receieve a message from the root for this period of time it will reset itsself.

The root node will send messages at set intervals defined in timerDelay. This should always be atleast two times the timerDelay.

Time in miliseconds.

◆ timerDelayRSSI

#define timerDelayRSSI   30000

The nodes gets assigned as root after this interval.

When a node doesn't receieve RSSI information from other nodes for this period of time it will assign itself as root.

Time in miliseconds.
