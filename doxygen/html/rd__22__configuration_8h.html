◆ timerDelayRoot

#define timerDelayRoot   40000

Reset the node after this interval.

When a node doesn't receive a message from the root for this period it will reset itself.

The root node will send messages at set intervals defined in timerDelay. This should always be at least two times the timerDelay.

Time in milliseconds.

◆ timerDelayRSSI

#define timerDelayRSSI   30000

The nodes gets assigned as root after this interval.

When a node doesn't receive RSSI information from other nodes for this period will assign themselves as root.

Time in milliseconds.
